1. Internet Technology: DNS Servers

The Domain Name System (DNS) translates human-readable domain names into machine-readable IP addresses. For example, www.google.com is resolved to an IP like 142.250.185.206. DNS uses a hierarchical lookup system from local, ISP, higher-level, and root servers to find addresses efficiently.

DNS simplifies browsing, allows websites to move servers without affecting users, and provides a scalable global addressing system.

Example: Entering www.onlinegames.co.uk sends a request to a DNS server that either resolves it or queries higher-level servers until the IP is returned, allowing the browser to connect to the web server.

2. Broadband Connection Technology

Broadband provides high-speed internet access for data transmission. Fiber-optic broadband uses light signals through glass fibers for extremely fast, low-latency, and reliable connections. This makes it ideal for cloud computing, streaming, and real-time communication.

3. Data Transmission

Data transmission moves bits from sender to receiver using various methods. It can be classified by direction (simplex, half-duplex, full-duplex), method (serial vs parallel), and synchronization (asynchronous vs synchronous). Key factors affecting method choice include distance, speed requirements, noise, latency, synchronization needs, hardware cost, and connector compatibility.

USB technology is a serial standard with four wires (VBUS, D–, D+, GND) allowing plug-and-play connection between host and peripherals. Benefits include backward compatibility, power delivery, and multiple transfer types; limits include cable length and latency.

Error checking ensures accurate data delivery. Methods include:

  • Parity bits (single or 2-D block) to detect/correct simple errors
  • Automatic Repeat Request (ARQ) with ACK/NAK retransmission
  • Checksum and echo checks to detect transmission errors
  • Advanced methods: CRC and FEC for robust detection/correction without retransmission

4. Cloud Computing & Web Applications

Cloud computing delivers computing services over the internet. Users access storage, databases, software, and processing power from remote servers rather than local devices. Cloud platforms like AWS, Azure, and Google Cloud provide scalable infrastructure with pay-as-you-go pricing.

Cloud benefits include cost efficiency, reliability, remote access, automatic updates, and rapid scalability. Limitations include dependence on internet connectivity and cybersecurity risks, requiring strong encryption, MFA, and access controls.

Web applications combine client-side technologies (HTML, CSS, JavaScript) with server-side processing (Python, PHP, SQL, APIs) to deliver interactive experiences. Virtual networks extend this connectivity logically, but require strong encryption and security practices.
